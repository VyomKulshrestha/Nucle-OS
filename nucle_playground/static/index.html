Why doesn't recovery probability improve with more redundancy on Illumina/Nanopore? Reed-Solomon parity here only reconstructs strands that were fully dropped. The current ternary decoder has zero tolerance for a strand that survived but was corrupted (a substitution/indel) — it's a real, documented gap (see the core repo's docs/architecture.md "Current status"), not a demo bug. Try the Twist or Pristine profiles (deletion-dominant / error-free) to see redundancy actually help.
